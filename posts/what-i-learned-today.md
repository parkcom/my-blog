클로드 코드와 함께 이 블로그를 만들면서, 웹 페이지를 이루는 세 가지 기술이 각각
어떤 일을 맡는지 정리해 봤습니다. 흔히 **HTML은 뼈대, CSS는 옷, JavaScript는 움직임**에
비유합니다.

## HTML — 구조(뼈대)

HTML은 페이지에 **무엇이 있는지**를 정의합니다. 제목, 문단, 목록, 버튼, 이미지 같은
콘텐츠의 의미와 순서를 담는 뼈대입니다.

- `<header>`, `<main>`, `<article>`, `<footer>` 같은 **시맨틱 태그**로 영역의 의미를 드러냅니다.
- 이 블로그에서는 글 목록과 글 본문이 들어갈 자리를 HTML이 잡아 둡니다.
- 의미에 맞는 태그를 쓰면 **접근성**과 검색에도 유리합니다.

> 핵심: HTML은 "내용이 무엇인가"를 다루지, 어떻게 보이는지는 다루지 않습니다.

## CSS — 표현(옷)

CSS는 HTML이 정의한 구조를 **어떻게 보이게 할지** 꾸밉니다. 색, 여백, 글꼴, 배치를
담당합니다.

- **커스텀 프로퍼티(변수)** 로 색을 한곳에서 정의해 라이트/다크 테마를 손쉽게 전환합니다.

  ```css
  :root {
    --bg: #ffffff;
    --text: #1a1a1a;
  }
  [data-theme="dark"] {
    --bg: #16161a;
    --text: #e6e6e9;
  }
  ```

- **미디어 쿼리**로 화면 크기에 따라 레이아웃을 바꿔 모바일에서도 깨지지 않게 합니다.
- 본문 폭과 줄 간격을 조절해 **읽기 편한 타이포그래피**를 만듭니다.

## JavaScript — 동작(움직임)

JavaScript는 페이지에 **행동과 상호작용**을 더합니다. 사용자의 클릭에 반응하고,
데이터를 불러오고, 화면을 갱신합니다.

- 마크다운(`.md`) 파일을 **`fetch`로 읽어 와** HTML로 변환해 화면에 그립니다.
- 테마 토글 버튼을 누르면 테마를 바꾸고, 선택을 **`localStorage`에 저장**해 다음 방문에도 유지합니다.
- 글 목록(`index.json`)을 읽어 카드로 렌더링하는 것도 JavaScript의 몫입니다.

## 셋이 함께 일하는 방식

| 기술       | 역할       | 비유  |
| ---------- | ---------- | ----- |
| HTML       | 구조·내용  | 뼈대  |
| CSS        | 표현·디자인 | 옷    |
| JavaScript | 동작·상호작용 | 움직임 |

세 기술은 **역할을 나눠 맡을 때** 가장 깔끔합니다. 구조는 HTML에, 모양은 CSS에,
동작은 JavaScript에 두면 코드가 이해하기 쉽고 고치기도 편합니다. 프레임워크 없이도
이 셋만으로 충분히 멋진 블로그를 만들 수 있다는 걸 오늘 배웠습니다.
