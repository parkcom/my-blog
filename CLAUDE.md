# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(및 다른 AI 에이전트)를 위한 가이드입니다.

## 프로젝트 개요

마크다운(`.md`) 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다.
글쓴이는 마크다운으로 글을 작성하고, 사이트는 이를 깔끔하고 읽기 좋은 웹 페이지로 렌더링합니다.

**핵심 목표**

- 깔끔하고 가독성 높은 디자인 (콘텐츠 중심, 넉넉한 여백, 편안한 타이포그래피)
- 라이트 / 다크 모드 지원
- 모바일 우선(반응형) 레이아웃
- 프레임워크·번들러 없이 순수 HTML, CSS, JavaScript(바닐라)만 사용

## 기술적 제약 (중요)

이 제약은 의도적으로 정한 것이므로 반드시 지켜주세요.

- **프레임워크 금지**: React, Vue, Svelte, Next.js 등을 도입하지 않습니다.
- **빌드 도구·번들러 금지**: Webpack, Vite, Rollup 등을 쓰지 않습니다. 브라우저에서 바로 실행되어야 합니다.
- **CSS 프레임워크 금지**: Tailwind, Bootstrap 등을 쓰지 않습니다. CSS는 직접 작성합니다.
- **표준 기술만 사용**: 모던 브라우저의 HTML5, CSS3, ES 모듈(ES Modules) 범위 안에서 구현합니다.
- 외부 라이브러리는 꼭 필요할 때만, 그리고 사용자와 합의한 뒤에만 추가합니다.
  - 마크다운 파싱이 필요해 라이브러리를 검토한다면 직접 구현 또는 단일 파일 라이브러리(예: CDN으로 불러오는 marked) 중 어느 쪽인지 먼저 상의해 주세요.

## 디자인 가이드라인

- **타이포그래피**: 본문 가독성을 최우선. 적절한 `line-height`(1.6~1.8), 본문 폭 제한(예: `max-width: 70ch`).
- **다크 모드**:
  - 색상은 CSS 커스텀 프로퍼티(`--변수`)로 정의하고, 라이트/다크 값을 한곳에서 토글합니다.
  - `prefers-color-scheme`로 시스템 설정을 기본값으로 따르되, 사용자가 수동 토글한 선택은 `localStorage`에 저장합니다.
  - 하드코딩된 색상 값(예: `#fff`)을 컴포넌트마다 흩뿌리지 말고 변수로 통일합니다.
- **반응형**: 모바일 우선으로 작성하고 `min-width` 미디어 쿼리로 확장합니다. 작은 화면에서 가로 스크롤이 생기지 않도록 합니다.
- **접근성**: 시맨틱 HTML 태그 사용, 충분한 색 대비, 키보드 포커스 스타일 유지, 이미지 `alt` 제공.

## 코드 컨벤션

- 들여쓰기는 스페이스 2칸.
- CSS는 커스텀 프로퍼티 + 의미 있는 클래스명을 사용합니다(BEM까지는 강제하지 않음).
- JavaScript는 ES 모듈(`<script type="module">`)을 사용하고, 전역 변수 오염을 피합니다.
- 주석은 "왜"를 설명할 때만 간결하게. 코드가 스스로 설명되도록 작성합니다.

## 프로젝트 구조

```
my-blog/
├── index.html          # 글 목록(홈). <head>에 FOUC 방지용 인라인 테마 스크립트 포함
├── post.html           # 개별 글 렌더링 페이지 (post.html?slug=<slug>)
├── css/
│   └── style.css       # 전역 스타일 + 라이트/다크 테마 변수
├── js/
│   ├── theme.js        # 다크 모드 토글 버튼 로직 (모든 페이지 공유)
│   ├── home.js         # posts/index.json을 읽어 글 목록 렌더
│   └── post.js         # slug로 .md fetch → marked로 변환해 렌더
└── posts/
    ├── index.json      # 글 매니페스트: [{ slug, title, date, summary }]
    └── *.md            # 블로그 글 원본(마크다운). 본문에 H1 제목은 넣지 않음(제목은 매니페스트가 출처)
```

### 핵심 설계 결정

- **마크다운 파싱**: `marked`를 버전 고정 CDN(ESM)으로 `post.js`에서 import해 사용. 직접 파서를 구현하지 않는다.
- **글 목록**: 빌드 단계가 없어 디렉터리 자동 스캔이 불가하므로 `posts/index.json` 매니페스트로 글과 메타데이터(제목·날짜·요약)를 관리한다.
- **FOUC 방지**: 각 HTML `<head>`의 인라인 동기 스크립트가 페인트 전에 `data-theme`를 확정한다(모듈은 defer라 늦어 깜빡임 발생). 이 한 조각만 ES 모듈이 아닌 인라인 스크립트인 이유다.
- **글 추가 절차**: `posts/<slug>.md` 작성 → `posts/index.json`에 항목 한 줄 추가.

## 개발·실행

빌드 단계가 없으므로 로컬 정적 서버로 바로 확인합니다. `file://`로 열면 ES 모듈/`fetch`가 막히므로 서버를 띄우세요.

```bash
# Python 내장 서버
python3 -m http.server 8000
# 또는
npx serve
```

브라우저에서 `http://localhost:8000` 접속.
