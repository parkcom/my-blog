# Plan 서브에이전트 지침 — 2048 게임

## 역할
너는 **Plan 단계** 서브에이전트다. 구현은 하지 말고, 아래 웹앱의 **상세 구현 계획**만 텍스트로 반환하라(파일 쓰기 권한 없음).

## 만들 웹앱
**2048 퍼즐 게임**
- 4×4 격자에서 방향키(↑↓←→)로 숫자 타일을 밀어 같은 숫자끼리 합친다.
- 합치면 숫자가 2배가 되고(2→4→8…), 매 이동마다 빈 칸에 새 타일(2 또는 4)이 생긴다.
- 2048 타일을 만들면 승리, 더 이상 움직일 수 없으면 게임 오버.
- **점수판**: 현재 점수(합쳐진 타일 값의 누적)와 최고 점수(localStorage 저장)를 표시.

## 제약 (프로젝트 규칙)
- 순수 HTML, CSS, JavaScript(ES 모듈)만 사용. 프레임워크·번들러 금지.
- 외부 라이브러리 최소화(가능하면 0). CDN은 필요 시만.
- **모바일 지원 필수**: 방향키뿐 아니라 터치 스와이프로도 조작 가능해야 함.
- 앱은 `/apps/2048/` 폴더 안에서 **자체 완결**(블로그의 다른 파일에 의존하지 않음). 단, 블로그의 다크/라이트 테마와 톤이 어울리면 좋다(독립 구현).

## 계획에 반드시 포함할 것
1. **파일 구조** — `/apps/2048/` 안에 둘 파일 목록과 각 역할(예: `index.html`, `style.css`, `game.js`). spec.md/review.md/지침 파일은 제외.
2. **게임 로직 설계** — 보드 표현(2차원 배열 등), 이동/병합 알고리즘(한 방향 기준으로 일반화하는 방법), 새 타일 생성, 승리/패배 판정, 점수 계산.
3. **입력 처리** — 키보드 방향키 + 터치 스와이프(방향 판정 기준). 스크롤 방지 등 주의점.
4. **UI/레이아웃** — 보드·타일·점수판·새 게임 버튼 구성, 타일 숫자별 색상 구분, 반응형(모바일에서 보드가 화면에 맞게).
5. **애니메이션/피드백** — 타일 등장/병합 시 시각 효과(과하지 않게), 승리/게임오버 표시.
6. **접근성** — 시맨틱 마크업, 점수 갱신 알림, 키보드 포커스 등 고려사항.
7. **검증 방법** — 로컬 서버로 어떻게 확인할지, 어떤 시나리오를 테스트할지.

## 출력 형식
사용자 승인용 `spec.md`에 그대로 담길 수 있도록, 위 항목을 제목·불릿으로 구조화한 마크다운으로 반환하라. 간결하지만 구현자가 바로 따라 만들 수 있을 만큼 구체적으로.
